= Contributions Use-Cases
:toc:
:toc-placement!:

This folder documents the **contributions route architecture** for the OSSN backend.
It defines expected behavior for contribution-related API flows before full implementation in the `app/contributions` package.

toc::[]

== Purpose

The goal of this folder is to capture contribution business and API behavior in an implementation-ready format.

These documents are the source of truth for:

* API route design (`/contributions/...` and project contribution paths)
* service-layer behavior in `app/contributions/service.py`
* repository interactions in `app/contributions/repository.py`
* status transitions, authorization checks, and error handling expectations

== Route Architecture Scope

The contributions use-cases describe expected route behavior for:

* `POST /contributions/projects/{project_id}/apply`
* `GET /contributions/me`
* `GET /contributions/projects/{project_id}`
* `PATCH /contributions/{contribution_id}`

The flow definitions here should remain aligned with the backend `app/contributions` domain package.

* Add one file per contributions use-case (e.g., `list-my-contributions.md`, `update-contribution-status.md`).
