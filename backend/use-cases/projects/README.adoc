= Projects Use-Cases
:toc:
:toc-placement!:

This folder documents the **projects route architecture** for the OSSN backend.
It defines expected behavior for project-related API flows before full implementation in the `app/projects` package.

toc::[]

== Purpose

The goal of this folder is to capture project business and API behavior in an implementation-ready format.

These documents are the source of truth for:

* API route design (`/projects/...`)
* service-layer behavior in `app/projects/service.py`
* repository interactions in `app/projects/repository.py`
* validation, filtering, and error handling expectations

== Route Architecture Scope

The projects use-cases describe expected route behavior for:

* `POST /projects`
* `GET /projects`
* `GET /projects/{project_id}`
* `GET /projects/help-wanted`

The flow definitions here should remain aligned with the backend `app/projects` domain package.

== Notes

* Add one file per projects use-case (e.g., `list-projects.md`, `get-project-details.md`).
* Each use-case should include actor, preconditions, main flow, alternative flows, and a Mermaid flowchart.
* Keep this folder focused on behavior and contracts; implementation details belong in backend code.
