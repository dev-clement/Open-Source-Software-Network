= Database Migrations
:toc:
:toc-placement!:

This directory contains the Alembic database migration configuration and versioned migration scripts for the OSSN backend.

toc::[]

== Purpose

Alembic manages incremental, versioned changes to the PostgreSQL database schema.
Each migration is a Python script that describes what changed and how to apply or reverse it.

== Structure

----
migrations/
  env.py              # Alembic runtime environment: wires settings and SQLModel metadata
  script.py.mako      # Template used to generate new migration files
  README.adoc         # This file
  versions/           # Generated migration scripts, one per revision
----

== How it works

- `env.py` imports the project database URL from `app.core.settings` and the SQLModel shared metadata from `app.db.models`.
- Alembic compares the current database schema against the metadata to detect changes.
- Autogenerate produces a revision file in `versions/` describing the diff.

== Common commands

All commands must be run from the `backend/` directory.

=== Create a new migration after changing a model

[source,bash]
----
alembic revision --autogenerate -m "describe your change here"
----

=== Apply all pending migrations

[source,bash]
----
alembic upgrade head
----

=== Roll back the last migration

[source,bash]
----
alembic downgrade -1
----

=== Check current migration state

[source,bash]
----
alembic current
----

=== Show full migration history

[source,bash]
----
alembic history --verbose
----

== Notes

- Alembic uses a synchronous DB driver (`psycopg2`) for migrations.
  The application itself uses `asyncpg` at runtime. Both are installed in the project.
- The database URL is never hardcoded. It is always read from environment variables via `app.core.settings`.
- Always review autogenerated migration files before applying them. Autogenerate is helpful but not perfect, especially for custom constraints, enums, or indexes.
- The `versions/` directory is committed to version control so schema history is shared across all environments.
