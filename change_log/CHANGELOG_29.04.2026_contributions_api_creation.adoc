== 2026-04-29

=== Added

* **Implemented `apply_to_project` endpoint in `backend/app/contributions/api.py` to allow users to apply to a project.**
  ** This endpoint assigns the project_id from the route to the contribution data and delegates the application logic to the contribution service.
  ** Returns HTTP 400 with error details if the service raises an exception.
* Centralized async DB session management in `backend/app/shared/db.py` and refactored API modules to use this shared dependency.
* Added `get_contribution_service` dependency with FastAPI's `Depends` for testability and maintainability.
* Added unit tests for `get_contribution_service` and `apply_to_project` (including error handling) in `backend/tests/test_contributions_api.py`.
* Improved documentation with detailed docstrings for new API methods.

* **Implemented `list_my_contributions` endpoint in `backend/app/contributions/api.py` to allow users to retrieve their own contributions.**
  ** This endpoint uses authentication to identify the user and delegates to the contribution service to fetch all contributions for that user.
  ** Returns HTTP 400 with error details if the service raises an exception.
  ** Added a detailed Python docstring explaining the endpoint's usage, arguments, and error handling.

* **Implemented `list_project_contributors` endpoint in `backend/app/contributions/api.py` to list all contributors for a given project.**
  ** This endpoint delegates to the contribution service to fetch all contributors for the specified project.
  ** Returns HTTP 400 with error details if the service raises an exception.

* Added unit tests for the `/me` endpoint (`list_my_contributions`) in `backend/tests/test_contributions_api.py`:
  ** Tests for valid user, invalid user, and HTTP 400 error handling.

* **Implemented `update_status` endpoint in `backend/app/contributions/api.py` to allow updating a user's contribution status for a project.**
  ** Handles ProjectNotFound and UserNotFound exceptions with proper HTTP error codes and messages.
  ** Returns HTTP 404 if project_id or user_id is not provided.
  ** Returns HTTP 400 for other service exceptions or validation errors.
  ** Added comprehensive unit tests for all error cases and parameter validation in `backend/tests/test_contributions_api.py`.
  ** Added detailed docstring for endpoint usage and error handling.
